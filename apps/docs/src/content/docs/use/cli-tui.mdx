The CLI and TUI are the fastest way to operate Carina on a local machine. Same runtime as JSON-RPC — different surface.

## Essentials

| Command | Purpose |
| --- | --- |
| `carina daemon start` | Start the local daemon |
| `carina doctor` | Health check (socket, kernel, tools) |
| `carina` / `carina run "…"` | Interactive TUI or one-shot task |
| `carina audit SESSION` | Replay decisions and effects |
| `carina audit verify SESSION` | Hash-chain integrity |
| `carina patch list \| rollback` | List / undo transactional patches |
| `carina call METHOD` | JSON-RPC helper for scripting |
| `carina workflow run NAME` | Run a workflow DAG |

## Typical day

<Steps>
  <Step title="Daemon up">
    ```bash frame="none"
    carina daemon start && carina doctor
    ```
  </Step>
  <Step title="Work in a repo">
    ```bash frame="none"
    cd /path/to/repo
    carina run "your task"
    # or: carina
    ```
  </Step>
  <Step title="Inspect / recover">
    ```bash frame="none"
    carina audit SESSION
    carina patch list SESSION
    ```
  </Step>
</Steps>

## Approvals

Under stricter profiles, the TUI prompts before high-risk effects. That is the capability kernel doing its job — see [Policy](/concepts/policy/).
