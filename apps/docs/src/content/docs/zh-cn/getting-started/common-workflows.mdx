## 第一次受治理会话

<Steps>
  <Step title="启动并检查">
    ```bash frame="none"
    carina daemon start
    carina doctor
    ```
  </Step>
  <Step title="在仓库里跑">
    ```bash frame="none"
    cd /path/to/your/repo
    carina run "Summarize the top-level layout of this repository"
    ```
  </Step>
  <Step title="核对轨迹">
    ```bash frame="none"
    carina sessions
    carina audit SESSION
    carina audit verify SESSION
    ```
  </Step>
</Steps>

## 修失败测试

```bash frame="none"
carina run "fix the failing test in parser.go and run the test"
```

## 回滚错误补丁

```bash frame="none"
carina patch list SESSION
carina patch rollback SESSION PATCH_ID
carina audit verify SESSION
```

## 查看 Agent 做了什么

```bash frame="none"
carina audit SESSION
carina report SESSION
carina items SESSION
```

## 跑审查工作流

```bash frame="none"
mkdir -p .carina/workflows
cp examples/workflows/review.json .carina/workflows/
carina workflow run review
```

## 嵌入 / 脚本

打开 [方法目录](/zh-cn/api/methods/) → 展开方法 → **Playground → Mock**，无需 daemon 也能看响应形状。

活 JSON-RPC 见 [会话 API](/zh-cn/api/sessions/) 示例或 SDK。

```bash frame="none"
carina gateway methods   # daemon/gateway 已启动时列出方法
```
