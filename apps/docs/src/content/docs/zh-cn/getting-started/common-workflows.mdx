## 第一次受治理会话

**目标：** daemon 起来、跑完一个任务、证明写过审计。

<Steps>
  <Step title="启动并检查">
    ```bash frame="none"
    carina daemon start
    carina doctor
    ```
  </Step>
  <Step title="在仓库里跑">
    ```bash frame="none"
    cd /path/to/your/repo
    carina run "Summarize the top-level layout of this repository"
    ```
  </Step>
  <Step title="核对轨迹">
    ```bash frame="none"
    carina audit SESSION
    carina audit verify SESSION
    ```
    将 `SESSION` 换成 TUI 或 CLI 输出中的 id。
  </Step>
</Steps>

**涉及：** daemon、agent 循环、审计链。

## 修失败测试

**目标：** 在 `safe-edit` 下改代码，尽量走补丁而非裸写。

```bash frame="none"
carina run "fix the failing test in parser.go and run the test"
```

若内核要求审批，对高风险命令是预期行为。先读审批原因，而不是关策略。

**涉及：** CommandExec 风险等级、PatchApply、[策略](/zh-cn/concepts/policy/)。

## 回滚错误补丁

**目标：** 撤销 Agent 编辑，而不手搓 Git 历史。

```bash frame="none"
carina patch list SESSION
carina patch rollback PATCH_ID
```

**涉及：** 事务补丁、[审计与回滚](/zh-cn/concepts/audit/)。

## 查看 Agent 做了什么

```bash frame="none"
carina audit SESSION
carina audit tail --last 20
carina audit verify SESSION
```

**涉及：** 哈希链、权限决策。

## 跑审查工作流

```bash frame="none"
mkdir -p .carina/workflows
cp examples/workflows/review.json .carina/workflows/
carina workflow run review
```

**涉及：** [工作流](/zh-cn/workflows/overview/)、子 Agent 衰减。

## 嵌入：调用 session.create

用 [会话 API](/zh-cn/api/sessions/) 示例，或打开 [方法目录](/zh-cn/api/methods/) → **Playground → Mock**，无需 daemon 也能看响应形状。

```bash frame="none"
carina call session.create
```

**涉及：** JSON-RPC、[嵌入路径](/zh-cn/api/overview/)。
