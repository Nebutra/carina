从干净安装到在本地仓库上跑通一次受治理的会话。 <Badge variant="accent">约 5 分钟</Badge>

<Steps>
  <Step title="启动 daemon">
    ```bash title="start-daemon.sh" showLineNumbers
    carina-daemon &
    # 或
    carina daemon start
    carina doctor # [!code highlight]
    ```

    daemon 负责会话、<Tooltip tip="由 Rust 能力内核按最小权限配置评估。">策略评估</Tooltip>与审计存储，并发现同目录下的 kernel 服务与原生工具。
  </Step>

  <Step title="进入工作区">
    ```bash frame="none"
    cd /path/to/your/repo
    ```
  </Step>

  <Step title="运行编码任务">
    ```bash title="run-task.sh" showLineNumbers
    carina run "Summarize the top-level architecture of this repo"
    # 或修复具体问题
    carina run "fix the failing test in parser.go" # [!code highlight]
    ```

    交互式 TUI：

    ```bash frame="none"
    carina
    ```

    会改变系统状态的工具调用都会经过能力内核；高风险操作可能要求审批。
  </Step>

  <Step title="检查审计轨迹">
    ```bash title="audit.sh" showLineNumbers
    carina audit SESSION
    carina audit verify SESSION
    carina audit tail --last 5
    ```

    每条记录参与哈希链，事后改写可被检测。
  </Step>

  <Step title="补丁与回滚">
    ```bash title="patch.sh" showLineNumbers
    carina patch list SESSION
    carina patch rollback PATCH_ID # [!code highlight]
    ```

    安全编辑的 diff 示意：

    ```go title="parser.go" showLineNumbers
    // before
    return err // [!code --]
    // after
    return fmt.Errorf("parse: %w", err) // [!code ++]
    ```
  </Step>
</Steps>

<Expandable title="doctor 失败怎么办？">
  运行 `carina doctor -v`，确认 `~/.carina/` 下存在 daemon 套接字。安装与 PATH 见 [安装](/zh-cn/getting-started/installation/)。
</Expandable>
