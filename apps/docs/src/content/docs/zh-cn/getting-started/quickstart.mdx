> **目标：** 在本地仓库上跑完一个任务，并留下可校验的审计轨迹。 <Badge variant="accent">约 5 分钟</Badge>

## 你将得到

- daemon 健康（`carina doctor`）  
- `carina run` 创建的 `safe-edit` 会话  
- 审计回放 + 哈希链校验  
- 若改了文件，如何列出 / 回滚补丁  

## 最快路径

<Steps>
  <Step title="安装并 doctor">
    未安装请先看 [安装](/zh-cn/getting-started/installation/)。

    ```bash frame="none"
    carina doctor
    ```

    需要显式 daemon：

    ```bash frame="none"
    carina daemon start
    carina status
    ```
  </Step>

  <Step title="在仓库里跑任务">
    ```bash title="run.sh"
    cd /path/to/your/repo
    carina run "Summarize the top-level architecture of this repo"
    ```

    当前 CLI 行为：在 `cwd` 创建 **`safe-edit`** 会话、提交任务并等待结束（`--background` 可仅排队后返回）。

    交互壳：

    ```bash frame="none"
    carina
    ```

    改变系统状态的工具调用会经过 <Tooltip tip="副作用前求值的十四种能力类型。">能力内核</Tooltip>；高风险可能要求审批。
  </Step>

  <Step title="查看审计">
    ```bash title="audit.sh"
    carina sessions
    carina audit SESSION
    carina audit verify SESSION
    carina report SESSION
    ```
  </Step>

  <Step title="补丁（若改了文件）">
    ```bash title="patch.sh"
    carina patch list SESSION
    carina patch show SESSION PATCH_ID
    carina patch rollback SESSION PATCH_ID
    ```
  </Step>
</Steps>

## 若失败

| 问题 | 处理 |
| --- | --- |
| `daemon unreachable` | `carina daemon start` · 检查 `~/.carina/daemon.sock` |
| 卡在审批 | `carina approve SESSION DECISION_ID` · 见 [策略](/zh-cn/concepts/policy/) |
| 模型/鉴权错误 | `carina auth list` · `carina auth login <provider>` |
