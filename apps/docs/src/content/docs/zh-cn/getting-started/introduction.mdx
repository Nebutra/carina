Carina 是**本地优先的 AI Agent 运行时**。它处在编程 Agent 离开对话、开始操作真实仓库的位置 — 策略门控、可校验审计、补丁可回滚。

## 为什么是 Carina

三句承诺，其余能力都挂在下面：

| 承诺 | 含义 |
| --- | --- |
| **先策略后副作用** | 十四种能力、配置、审批 — 模型不能绕过内核 |
| **哈希链审计** | 追加写事件；`carina audit verify` 可验伪 |
| **事务回滚** | 文件变更走补丁与出处，而非静默覆盖 |

> Agent 全力执行。你保持控制。权威在本机。

## 三步上手

<Steps>
  <Step title="安装">
    ```bash frame="none"
    curl -fsSL https://carina.nebutra.com/install.sh | sh
    ```
    更多方式：[安装](/zh-cn/getting-started/installation/)。
  </Step>
  <Step title="跑一个任务">
    ```bash frame="none"
    carina daemon start && carina doctor
    cd /path/to/your/repo && carina run "Summarize this repo"
    ```
  </Step>
  <Step title="证明受治理">
    ```bash frame="none"
    carina audit SESSION && carina audit verify SESSION
    ```
  </Step>
</Steps>

完整导览：[快速上手](/zh-cn/getting-started/quickstart/) · 配方：[常用工作流](/zh-cn/getting-started/common-workflows/)。

## 面向谁

- 希望 Agent 在**本机仓库**上干活且可审计/回滚的工程师  
- 搭建**内部 agent runner** 的平台团队  
- 关注文件、shell、密钥的安全向团队  
- 在 IDE / TUI / CI / Web 后嵌入运行时的工具作者  

**不**主要面向：成品编辑器聊天机器人，或以托管云 Agent 为核心的产品。

## 表面

| 表面 | 角色 |
| --- | --- |
| **CLI / TUI** | 本地交互编码与运维 |
| **JSON-RPC + Gateway** | 嵌入你的产品 |
| **SDKs** | TS / Python / Go |
| **Workers** | 带租约的分布式执行 |
| **MCP** | 外部工具，仍受策略门控 |

同一引擎：无论从哪附着，都是 session + profile + audit。
