## 你将得到

- `PATH` 上的 `carina` CLI  
- 可启动本地 daemon（`carina daemon start`，或 `carina` 自动拉起）  
- 可选 workers 与客户端表面（TUI 内置于 CLI）  

## 前置

- **macOS 13+** 或现代 **Linux**（x86_64 / arm64）  
- 仅在拉包或调云端模型时需要网络（运行时本身本地优先）  

## 安装

需要完整工具链时优先 **shell 安装器**。

<Tabs>
  <Tab label="Shell">
    ```bash title="install.sh" frame="none"
    curl -fsSL https://carina.nebutra.com/install.sh | sh
    ```
  </Tab>
  <Tab label="Homebrew">
    ```bash title="brew" frame="none"
    brew install nebutra/tap/carina
    ```
  </Tab>
  <Tab label="npm">
    ```bash title="npm" frame="none"
    npm install -g @nebutra/carina
    ```
  </Tab>
  <Tab label="源码">
    ```bash title="install-from-source.sh"
    git clone https://github.com/nebutra/carina.git
    cd carina
    make install
    ```
  </Tab>
</Tabs>

## 验证

<Steps>
  <Step title="版本">
    ```bash frame="none"
    carina version
    ```
  </Step>
  <Step title="初始化（可选）">
    ```bash frame="none"

    ```
  </Step>
  <Step title="Doctor">
    ```bash frame="none"

    ```

  </Step>
</Steps>

## Daemon

| 命令 | 含义 |
| --- | --- |
| `carina daemon start` | 启动本 CLI 拥有的本地 daemon |
| `carina daemon status` | 状态（不自动启动） |
| `carina daemon stop` | 仅停止本 CLI 拥有的 daemon |
| `carina status` | 健康与计数 |

交互式 `carina`（TUI）会在需要时**自动启动** daemon。默认套接字：`~/.carina/daemon.sock`。
