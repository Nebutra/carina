会话绑定对话状态、**能力配置**与**工作区根目录**。工作区在 `session.create` 时通过 `workspace_root` 绑定 — 没有单独的注册方法。

<Update label="文档" date="2026-07-17">
  方法目录含 **Playground**（Mock / Live）与 Markdown 导出（`index.md`），便于 LLM 工作流。
</Update>

## 生命周期

```
create → (pause / resume) → close
         ↘ submit tasks / stream events
         ↘ fork / checkpoint / goal
```

## `session.create`

<span class="docs-method-meta">
  <Badge variant="accent">write</Badge>
  <Badge>local socket</Badge>
</span>

创建绑定工作区与权限配置的会话。

<ApiReference>
  <p>
    <strong>参数</strong>
  </p>
  <ParamField path="workspace_root" type="string" required body>
    工作区目录的绝对路径。会话存活期间保持绑定。
  </ParamField>
  <ParamField path="profile" type="string" body default="safe-edit">
    权限配置名（如 `safe-edit`、`read-only`）。由能力内核评估。
  </ParamField>

  <p>
    <strong>结果</strong>
  </p>
  <ResponseField name="session_id" type="string" required>
    不透明会话 id，供后续 `session.*` / `task.*` 调用使用。
  </ResponseField>
  <ResponseField name="workspace_id" type="string" required>
    绑定的工作区 id。
  </ResponseField>
  <ResponseField name="profile" type="string" required>
    生效的能力配置。
  </ResponseField>

## 核心方法

| 方法 | 用途 |
| --- | --- |
| `session.create` | 创建会话（工作区 + 配置） |
| `session.get` / `session.list` | 元数据与列表 |
| `session.pause` / `session.resume` | 挂起 / 继续 |
| `session.close` | 终止 |
| `session.replay` | 回放原始审计事件流 |
| `session.items` | UI/SDK 用归一化 item 流 |
| `session.review` | 治理投影 |
| `session.events.stream` | 实时通知 |

JSONL / SQLite 导出由 `audit.export` 提供，不是 `session.*`。

## 任务操作

| 方法 | 用途 |
| --- | --- |
| `task.submit` | 提交 prompt/任务 |
| `task.cancel` | 取消运行中任务 |
| `task.status` | 查询状态 |
| `task.action.approve` / `task.action.deny` | 解决待审批 |

任务事件通过 `session.events.stream` 消费。
