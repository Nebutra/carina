From a clean install to a governed session on a local repo.

<Steps>
  <Step title="Start the daemon">
    ```bash title="bash"
    carina-daemon &
    # or
    carina daemon start
    carina doctor
    ```

    The daemon owns sessions, policy evaluation, and the audit store. It discovers the kernel service and native tools next to its binary.
  </Step>

  <Step title="Open a workspace">
    ```bash title="bash"
    cd /path/to/your/repo
    ```
  </Step>

  <Step title="Run a coding task">
    ```bash title="bash"
    carina run "Summarize the top-level architecture of this repo"
    # or fix something concrete
    carina run "fix the failing test in parser.go"
    ```

    Interactive TUI:

    ```bash title="bash"
    carina
    ```

    Tool calls that mutate state hit the capability kernel. Depending on profile, high-risk actions may prompt for approval.
  </Step>

  <Step title="Inspect the audit trail">
    ```bash title="bash"
    carina audit SESSION
    carina audit verify SESSION
    carina audit tail --last 5
    ```

    Each entry participates in a hash chain so later rewrite of history is detectable.
  </Step>

  <Step title="Patches and rollback">
    ```bash title="bash"
    carina patch list SESSION
    carina patch rollback PATCH_ID
    ```
  </Step>
</Steps>
