From a clean install to a governed session on a local repo. <Badge variant="accent">5 min</Badge>

<Steps>
  <Step title="Start the daemon">
    ```bash title="start-daemon.sh" showLineNumbers
    carina-daemon &
    # or
    carina daemon start
    carina doctor # [!code highlight]
    ```

    The daemon owns sessions, <Tooltip tip="Least-privilege profiles evaluated by the Rust capability kernel.">policy evaluation</Tooltip>, and the audit store. It discovers the kernel service and native tools next to its binary.
  </Step>

  <Step title="Open a workspace">
    ```bash frame="none"
    cd /path/to/your/repo
    ```
  </Step>

  <Step title="Run a coding task">
    ```bash title="run-task.sh" showLineNumbers
    carina run "Summarize the top-level architecture of this repo"
    # or fix something concrete
    carina run "fix the failing test in parser.go" # [!code highlight]
    ```

    Interactive TUI:

    ```bash frame="none"
    carina
    ```

    Tool calls that mutate state hit the capability kernel. Depending on profile, high-risk actions may prompt for approval.
  </Step>

  <Step title="Inspect the audit trail">
    ```bash title="audit.sh" showLineNumbers
    carina audit SESSION
    carina audit verify SESSION
    carina audit tail --last 5
    ```

    Each entry participates in a hash chain so later rewrite of history is detectable.
  </Step>

  <Step title="Patches and rollback">
    ```bash title="patch.sh" showLineNumbers
    carina patch list SESSION
    carina patch rollback PATCH_ID # [!code highlight]
    ```

    Diff view of a safe edit:

    ```go title="parser.go" showLineNumbers
    // before
    return err // [!code --]
    // after
    return fmt.Errorf("parse: %w", err) // [!code ++]
    ```
  </Step>
</Steps>

<Expandable title="What if doctor fails?">
  Run `carina doctor -v` and confirm the daemon socket exists under `~/.carina/`. See [Installation](/getting-started/installation/) for PATH and binary layout.
</Expandable>
