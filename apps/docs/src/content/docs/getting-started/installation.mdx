Carina ships as a local toolchain: a CLI, a long-running daemon, optional workers, and client surfaces (TUI, IDE, web operator).

## Prerequisites

- macOS 13+ or modern Linux (x86_64 / arm64)
- Network access only if you pull models or packages (runtime itself is local-first)

## Install

Pick one channel. Prefer the shell installer when you want the full toolchain.

<Tabs>
  <Tab label="Shell">
    ```bash title="install.sh" frame="none"
    curl -fsSL https://get.carina.dev | sh
    ```
  </Tab>
  <Tab label="Homebrew">
    ```bash title="Brewfile" frame="none"
    brew install nebutra/tap/carina
    ```
  </Tab>
  <Tab label="npm">
    ```bash title="package.json" frame="none"
    npm install -g @nebutra/carina
    ```
  </Tab>
  <Tab label="From source">
    ```bash title="install-from-source.sh"
    git clone https://github.com/nebutra/carina.git
    cd carina
    make install
    ```
  </Tab>
</Tabs>

## Verify

```bash title="verify.sh"
carina version
carina doctor
```

`carina doctor` checks daemon reachability, config paths, and common misconfigurations.

<Callout variant="note">
  Package names and install URLs may track the open-source release channel. Prefer `carina doctor` after any upgrade.
</Callout>
