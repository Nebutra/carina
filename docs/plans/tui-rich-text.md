# TUI Rich Text And Math Rendering

Status: planned → in progress
Owner: TUI
Prior art surveyed: Claude Code (marked lexer + custom formatToken → Ink nodes,
cli-highlight/highlight.js with syntect NAPI fast path, OSC 11 theme detection,
no LaTeX) and OpenAI Codex `codex-rs/tui` (pulldown-cmark + ~2.8k-line custom
event writer, syntect + two-face, URL-aware textwrap, append-only source with
stable/tail streaming commit, table holdback, no LaTeX).

## Problem

The transcript is a plain-text, line-truncated event log:

- No markdown rendering: headings, emphasis, lists, code blocks, tables, and
  links render as raw text (`go/tui/transcript.go` renders manually assembled
  strings; no goldmark/glamour/chroma dependency exists).
- No soft word wrap: overflow is ellipsis-clipped by `ansi.Truncate`
  (`transcript.go` fitLine, `view.go` fitRenderedLine), so long prose is cut at
  the right edge instead of reflowed.
- No math rendering of any kind.
- Free-form assistant prose is intentionally suppressed in favor of structured
  action summaries; final-response prose therefore has no rich display path.

## Non-Negotiable Invariants

- `sanitize()` keeps stripping ANSI/control sequences from all inbound text.
  Styling is generated by the renderer from sanitized source; raw model bytes
  never reach the terminal as escape sequences.
- All styling goes through semantic theme roles (`go/tui/theme`). No hardcoded
  colors. The Mono profile must degrade to readable plain text.
- Deterministic rendering: same source + same width → same lines. Rendering
  stays a pure function suitable for golden tests and the existing `View()`
  render-regression benchmark.
- CJK correctness: all width math uses the existing runewidth/uniseg path.
- Chain-of-thought stays hidden. Rich rendering applies to user-facing
  assistant output (final responses, plan text), not to suppressed internals.

## Milestones

### P0 — Soft word wrap (infrastructure)

Replace truncation with span-aware soft wrap for prose content:

- New `go/tui/wrap.go`: wrap styled lines to a target width with
  initial/subsequent indent support; grapheme- and CJK-width-aware.
- URL-aware: URL-like tokens never break mid-token (mirrors Codex
  `wrapping.rs`); pair with OSC 8 hyperlinks emitted by the renderer (never
  from inbound text).
- Structured rows (tables of the event log, diff hunks, status lines) keep
  clipping; only prose bodies wrap. Resize re-wraps from source via the
  existing `resizePresentations()` hook.

### P1 — Markdown pipeline

- Parser: `goldmark` (CommonMark + GFM tables/strikethrough only). Renderer: a
  custom AST walker in `go/tui/markdown/` producing styled lines against new
  theme roles: `RoleHeading`, `RoleCodeInline`, `RoleCodeBlock`, `RoleLink`,
  `RoleListMarker`, `RoleBlockquote`, `RoleTableBorder`, `RoleMathApprox`.
  Glamour is deliberately not used: its own theming would bypass the
  design-token role system and width-aware table layout needs local control.
- Fence unwrapping: conservatively unwrap ```` ```markdown ```` fences that
  contain a table header+delimiter pair (mirrors Codex `unwrap_markdown_fences`).
- Surface final assistant prose through this renderer; action summaries remain
  the default row presentation.

### P2 — Streaming, highlighting, tables

- Streaming model (mirrors Codex `streaming/controller.rs`): keep the
  append-only sanitized source per message; on each commit re-render the whole
  source, split into a stable region (appended transcript entries, immutable)
  and a mutable tail (a keyed `pushPresentation` entry replaced in place).
  Newline-gated commits: an incomplete trailing line is never rendered.
- Holdback: a detected table (header + delimiter) and any unterminated code
  fence or math block stay in the mutable tail until closed or the stream ends.
- Code blocks: `chroma` highlighting mapped through theme roles, with
  guardrails (skip highlighting above 512 KB or 10k lines; fall back to plain
  `RoleCodeBlock`). Language from the first info-string token.
- Tables: width-aware column sizing with a key/value transposition fallback
  when the width budget cannot fit columns (mirrors Codex table pipeline).

### P3 — Math (Unicode approximation)

Neither Claude Code nor Codex renders math; both share a tiered-degradation
house style, and the terminal-native baseline tier is implemented here:

- Detect `$...$`, `$$...$$`, `\(...\)`, `\[...\]` spans with a math holdback
  (a partial formula is never rendered).
- Deterministic TeX-subset → Unicode transform in `go/tui/mathapprox/`:
  superscripts/subscripts (`x^2` → `x²`), Greek letters, `\frac{a}{b}` → `a⁄b`,
  `\sqrt` → `√`, common operators/arrows/relations. Pure function, golden
  tested.
- Fallback: any unrecognized construct renders verbatim styled as
  `RoleMathApprox`, visually delimiting the formula without corrupting it.
- Image-protocol rendering (Kitty/iTerm2/sixel) is a non-goal, consistent with
  the roadmap's terminal-capability boundary.

## Acceptance

- `go test ./go/tui/...` including new golden tests for wrap, markdown, table
  layout, holdback state machine, and math approximation; CJK and Mono-profile
  cases included.
- `View()` render-regression benchmark stays within budget with markdown
  rendering enabled on a workspace-sized transcript.
- No new escape-sequence path from inbound text to the terminal (sanitize
  boundary unchanged).
