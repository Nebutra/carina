# Carina 0.6.2

Carina 0.6.2 is an agent-runtime, orchestration, and operator UX release. It
replaces the previously occupied `v0.6.1` release line; do not retag or
overwrite 0.6.1.

Highlights:

- global Agent View with needs-input, working, and completed rosters;
- governed worktree isolation and durable checkpoint history;
- a production Bubble Tea TUI with transactional composer ownership, semantic
  keybindings, checkpoint rewind, scoped history search, background attention,
  normal-buffer/transcript escape hatches, and real PTY regression coverage;
- complete authored TUI and bootstrap copy for English, Simplified Chinese,
  Japanese, Korean, Spanish, and French, backed by locale parity, placeholder,
  plural, and register-lint contracts;
- crash-safe workflow, channel, and restore journals;
- explicit channel incident reconciliation with no automatic replay;
- artifact inspection and model usage/cost operator surfaces;
- actionable doctor findings for interrupted workflows, restores, and channels;
- typed runtime initialization, protocol discovery, and richer client surfaces.

Agent runtime and orchestration:

- streaming Agent Swarm DAGs with structured and conditional edges, dynamic
  generator nodes, bounded live inter-node messages, real remote-worker
  placement, hierarchical budget aggregation, and progress rollups;
- declarative subagent tool/spawn restrictions enforced by a dedicated kernel
  capability, plus opt-in best-of-N patch generation with independent
  verification and write-provenance protection;
- image content blocks carried as content-addressed `MediaRef` artifacts and
  delivered only to models that explicitly advertise image input support;
- token-triggered multi-tier compaction with tail-signal preservation,
  verbatim-user retention, versioned receipts, and deterministic key-file
  tracking;
- MCP-scoped `mcp_find` tool discovery with schemas returned on demand;
- managed-locked configuration keys, tri-level extension enable policy, and
  versioned state envelopes that quarantine unsupported future data rather
  than deleting it.
- context-aware Skill prompts with bounded metadata discovery, explicit
  `$name` loading, collision-safe slash-command fallback, and opt-in exact
  implicit triggers that never grant capabilities.
- pinned direct compilation for all six Zig tools, including macOS 26 local
  builds that target the documented macOS 13 minimum.

Reliability and developer experience:

- idempotent task submission IDs across the CLI and Go, Python, and TypeScript
  SDKs, with durable acknowledgement recovery and command fences that prevent
  late completions from mutating a newer session generation;
- context-aware keymap validation that protects normal text input, keeps pager
  controls scoped to their active surface, supports multi-step chords and
  literal Escape/Enter capture, rejects duplicate JSON actions, and hot-reloads
  only validated snapshots;
- canonical event cursors, governed provider retries with typed errors and
  backoff, consecutive-failure circuit breaking, and process containment
  negotiation for workers;
- authoritative tool-call lifecycle events, cancellation propagation across
  model, command, MCP, approval, and structured-question waits, and bounded
  artifact storage with lifecycle metrics and garbage collection;
- synchronized Go, Python, and TypeScript operator APIs, VS Code/web smoke
  coverage, Linux and macOS archives, native npm package assembly, and
  fail-closed Homebrew/signing/notarization release automation;
- complete 12-binary native npm packages including Headroom, offline global-install smoke tests,
  immutable draft-release npm bundles for retry recovery, and monotonic
  serialized Homebrew tap updates;
- immutable public GitHub Release assets: reruns verify the exact 32-asset set
  (native archives, Linux packages, Windows workers, operator clients,
  installer, npm bundle, checksums, and Apple evidence), and Homebrew consumes
  published checksums rather than rebuilt bytes;
- a local release preflight that emits machine-readable `PASS`, `FAIL`,
  `BLOCKED`, and `SKIP` results instead of treating absent external credentials
  as success.

Operator commands added or stabilized:

```text
carina session review <session_id>
carina channel pending
carina channel reconcile <sender_id> <event_id> <executed|not-executed> --yes
carina artifact stat <session_id> <artifact_id>
carina artifact read <session_id> <artifact_id>
carina cost [session_id]
```

Install and upgrade:

```bash
brew install Nebutra/tap/carina
brew update && brew upgrade carina
```

The release workflow also prepares Linux `arm64`/`amd64` archives and the
`@nebutra/carina` launcher with four native platform packages. Their first
public npm publication remains blocked on npm package bootstrap and trusted
publisher configuration; this note does not claim they are already available.

Channel reconciliation is deliberately manual. Verify whether the external
side effect executed before choosing an outcome; the runtime will not silently
replay an ambiguous permission or external event.
