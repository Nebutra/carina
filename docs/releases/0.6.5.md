# Carina 0.6.5

Patch release on the 0.6 line. Focus: safe, understandable continuity after
an agent session is interrupted.

## Highlights

### Production-grade session continuity

- Abandoned `running` tasks are reconciled into durable `interrupted` state
  instead of being presented as live work.
- Safe checkpoints resume automatically only after execution fencing, tool
  effect, workspace drift, and external side-effect proofs all pass.
- Automatic recovery is limited to one attempt per interruption generation.
  Explicit operator cancellation is never automatically undone.
- Local and remote execution generations reject stale terminal publishers;
  terminal status, summary, and patch lineage commit atomically.

### Honest recovery evidence

- Tool calls persist a runtime-authoritative effect contract. Unknown,
  non-idempotent, ambiguous workspace, and unkeyed external effects fail
  closed to review.
- Checkpoints bind read dependencies, committed patch targets, and patch
  lineage to a workspace anchor. Drift, symlink escape, inaccessible files,
  and hashing-budget violations block automatic recovery.
- Lost provider responses expose billing uncertainty rather than claiming
  external exactly-once behavior.

### Continuation UX and SDKs

- `/resume` prioritizes sessions needing attention and distinguishes
  interrupted recovery from active execution.
- Interrupted task status is localized in English, Simplified Chinese,
  Japanese, Korean, Spanish, and French.
- Go, TypeScript, and Python SDKs expose typed continuity, interruption,
  recovery, execution lease, and workspace anchor fields.

## Upgrade notes

- Restart long-running `carina-daemon` processes after upgrading so the new
  runtime epoch and state-directory lease become authoritative.
- Legacy checkpoints remain readable but cannot auto-resume without a new
  workspace anchor; they are shown as requiring review.
- Standalone installs can use `carina update`. Homebrew installs use
  `brew update && brew upgrade carina` after the release workflow updates the
  official tap.
