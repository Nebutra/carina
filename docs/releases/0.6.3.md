# Carina 0.6.3

Carina 0.6.3 is a product HITL / TUI close-out release on the 0.6 line. It
ships the remaining repository-owned agent-shell gaps after 0.6.2 without
changing the dual-axis permission model or silent-YOLO brand boundary.

## Highlights

### Product HITL (Waves L–M)

- Product modes: `ask` \| `always-approve` \| `dont-ask` \| `accept-edits`
- `accept-edits` auto-allows only `FileWrite` / `PatchApply`; shell, network,
  and secrets still prompt
- Session/project **path prefix grants** for file capabilities, with a
  dangerous-path / dangerous-command denylist that never auto-reuses stored
  grants
- Subagent permission inheritance contract (profile attenuate, session axis
  copy, product HITL daemon-global)

### Plan review

- `/view-plan` overlay: approve / request changes / quit
- Line and range comments (`c`, `m`) seed the request-changes composer draft

### Context compact

- `session.checkpoint.compact` at **idle** task boundaries (paused, completed,
  failed, …), not only paused
- Still refuses mid-execution; live agent loops continue to compact in-memory
  transcripts independently
- Auto-compact at ≥85% when `context.summary` reports compact available

### Side dual-pane

- `/side` and `/btw --fork` freeze the main transcript and show a dual-pane
  layout when the terminal is wide enough
- `/side-close` returns to the main session

### Quality / i18n

- Traditional Chinese (`zh-Hant`) derived catalogs with CI freshness checks
- Docs site and protocol catalog alignment for the four product HITL modes

## Upgrade notes

- Restart `carina-daemon` after installing so workers pick up the new binary
- Existing exact approval grants remain valid; new file session/project
  approvals also install a safe directory prefix companion
- `/resume <task_id>` remains a compatibility alias; prefer `/task-resume`

## Skipped (external / human)

- Native zh-Hant editorial review, IME terminal matrix, ACP / marketplace /
  silent YOLO, Apple notary / npm publisher ops as external activation gates
