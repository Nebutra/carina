# Carina 0.6.4

Patch release on the 0.6 line. Focus: local interactive UX — you should not
have to start `carina-daemon` by hand just to open the TUI.

## Highlights

### Auto-start daemon from `carina-tui`

- Bare `carina` already auto-started the daemon when the socket was down.
- `carina-tui` only dialed the socket, so operators had to run `carina-daemon`
  separately.
- **0.6.4** shares spawn/ensure in `go/localdaemon` and calls it from
  `carina-tui` before connect. Same ownership record as the CLI so
  `carina daemon stop` still works.

### Docs polish

- Expressive Code cards follow the site theme (light/dark) instead of
  always-dark chrome.

## Upgrade notes

- Restart any long-running `carina-daemon` after install (or just open
  `carina` / `carina-tui` and let auto-start pick up the new binary).
- Standalone installs: `make install` from the source tree, or
  `carina update` once this tag's GitHub Release assets are published.
